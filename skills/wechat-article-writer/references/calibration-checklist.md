# Article Calibration Checklist

文章输出前的最终校准清单。**每一项都必须检查通过**。

---

## 🔴 红线检查（任何一项失败 = 退回重写）

### 信息真实性

- [ ] **无编造数据** — 没有"我测试了 X 个"但实际未测试的内容
- [ ] **无虚假百分比** — 所有百分比有来源（如"幻觉率降 26.8%"需有官方来源）
- [ ] **无杜撰引用** — 所有引用有真实人名 + 来源链接
- [ ] **无"研究表明"无来源** — 所有"据 XX 研究"都标注具体研究名称和链接

### 运营内容清理

- [ ] **无"获取方式"** — 删除"关注公众号回复 XXX"
- [ ] **无"下期预告"** — 删除"下期预告：《XXX》"
- [ ] **无引导互动** — 删除"点个在看"、"评论区留言"

---

## 🟡 重要检查（3 项以上失败 = 退回修改）

### 来源标注

- [ ] 每个数据点有来源链接或引用
- [ ] 每个"XX 表示"有真实人名
- [ ] 每个"据 XX 报道"有媒体名称
- [ ] 无匿名"专家表示"、"业内人士称"

### 内容质量

- [ ] 无 AI 套话（"震惊"、"炸了"、"细思极恐"）
- [ ] 无空洞预测（"必将"、"一定会"、"注定"）
- [ ] 无排比句堆砌
- [ ] 无 emoji 滥用

### 写作风格

- [ ] 第一段直接说发生了什么（无铺垫）
- [ ] 每段有信息增量（删除测试不损失信息）
- [ ] 观点有证据支撑（每判断至少 1 数据点）
- [ ] 结尾有力（无"未来会怎样"空话）

---

## 🟢 结构多样化检查（NEW — 减少 AI 味儿）

### Structure Match

- [ ] 结构与选题类型匹配（新闻→A，评测→B，经验→C，工具→D）
- [ ] 无固定"一、二、三、四、五"章节结构（除非必要）
- [ ] 小标题数量合理（0-3 个，不是每段都有）
- [ ] 段落长度有变化（不是全部 2-4 句）

### Markdown Usage

- [ ] 无过度 Markdown（标题/表格/代码块只在必要时使用）
- [ ] 表格只在比较 3+ 项目时使用
- [ ] 代码块只用于多行代码（不是单个命令）
- [ ] 无每段都加粗的情况

### Human Touch

- [ ] 有口语化表达（"说实话"、"但"、"其实"）
- [ ] 有不确定性表达（"可能"、"感觉"）
- [ ] 有情绪变化（不是全程冷静客观）
- [ ] 有待补充清单（2-4 项，引导用户加个人内容）

---

## 🟢 优化建议（可选，但建议改进）

### 信息密度

- [ ] 无冗余解释（读者已知常识不解释）
- [ ] 无重复表述（同一观点不说两遍）
- [ ] 表格≤3 列（移动端可读性）

### 可读性

- [ ] 段落短小（2-4 句为主）
- [ ] 小标题清晰（一眼知道这节讲什么）
- [ ] 颜色标注恰当（不滥用彩色框）

---

## 校准流程

```
1. 文章生成后，自动运行此清单检查
   ↓
2. 红线检查有任何失败 → 退回重写
   ↓
3. 重要检查 3 项以上失败 → 退回修改
   ↓
4. 全部通过后，输出文章 + 来源清单
   ↓
5. 用户可手动添加运营元素（如需要）
```

---

## 来源清单格式

文章输出时，同时输出来源清单：

```markdown
---
## 来源清单

1. [OpenAI 官方博客](https://openai.com/index/xxx) — GPT-5.4 发布信息
2. [Hacker News 讨论](https://news.ycombinator.com/item?id=xxx) — 用户反馈
3. [XX 媒体报道](https://xxx.com/xxx) — 第三方观点
---
```

---

*此清单应自动执行，但用户也可手动复核。*
